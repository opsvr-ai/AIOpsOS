## 对话安全确认与信息收集机制：智能人机协作面板

针对敏感操作确认和用户信息收集的需求，我们将设计一套**内联交互组件**，让智能体在执行敏感操作或需要补充信息时，在对话流中直接生成安全确认卡片或参数收集表单，用户无需离开对话页面即可完成决策。

---

### 一、核心概念：两类内联交互组件

#### 1. 安全确认卡片 (Security Confirmation Card)
- **触发场景**：智能体生成的分析代码或操作指令被安全沙箱/AST分析判定为**高风险**（如涉及`rm -rf`、修改系统配置、重启服务等）。
- **交互模式**：阻断式确认，用户必须明确选择“确认执行”或“拒绝”才能继续流程。

#### 2. 参数收集表单 (Inline Parameter Form)
- **触发场景**：智能体发现**信息缺失**，需要用户补充参数才能继续（如“请选择要巡检的主机组”、“请指定时间范围”）。
- **交互模式**：非阻断式，用户填写后提交，智能体收到补充信息后继续。

---

### 二、安全确认卡片设计

#### 外观与位置
- 嵌入对话流，作为一条特殊的“智能体消息”出现。
- 顶部醒目标识：`⚠️ 安全警告`（橙色）或 `🚫 高风险操作`（红色）。
- 卡片内容区展示：
  - **风险说明**：自然语言描述（如“该操作将重启生产环境服务 pay-01，可能导致短暂服务中断”）。
  - **代码预览**：被标记为风险的代码段（语法高亮，折叠显示，点击“查看代码”展开）。
  - **影响范围**：涉及的主机、服务、预计影响时长。
- 底部两个按钮：
  - `[确认执行]` 主按钮（红色或橙色，明确告知危险）。
  - `[拒绝执行]` 次按钮（灰色）。
- 可选：增加“记住我的选择”复选框（针对同类型操作，未来自动确认/拒绝）。

#### 交互细节
```jsx
<SecurityConfirmCard>
  <Header icon="⚠️" title="安全警告：高风险操作" />
  <Content>
    <p>该操作将执行以下命令：</p>
    <CodeBlock code="sudo systemctl restart pay-01" language="bash" />
    <p>影响范围：主机 pay-01 (10.2.3.15)，预计暂停服务 5-10 秒。</p>
    <p>风险等级：<Tag color="red">高风险</Tag></p>
  </Content>
  <Footer>
    <Checkbox>记住我的选择（30分钟内生效）</Checkbox>
    <ButtonGroup>
      <Button danger>确认执行</Button>
      <Button>拒绝执行</Button>
    </ButtonGroup>
  </Footer>
</SecurityConfirmCard>
```

#### 与后端流程对接
- 智能体调用工具（或生成代码）时，工具执行前由沙箱分析风险等级。
- 若风险等级达到阈值，工具不执行，而是通过 `human_interrupt_node` 挂起会话。
- 将风险详情通过 WebSocket 推送到前端，前端渲染安全确认卡片。
- 用户选择后，通过 `/api/sessions/{id}/intervention-response` 将结果回传，恢复执行。

### 三、参数收集表单设计

#### 外观与位置
- 同样嵌入对话流，出现在智能体规划确定需要更多信息之后。
- 标题：`📋 需要补充信息`。
- 表单区：根据所需参数动态生成表单字段，可能包含：
  - 下拉选择（如主机列表、场景模板）。
  - 日期/时间选择器（如巡检时间范围）。
  - 文本输入（如阈值、备注）。
  - 单选/多选框（如通知方式）。
- 底部按钮：`[提交]` 或 `[跳过/使用默认值]`。

#### 交互细节
```jsx
<InlineParameterForm>
  <Header icon="📋" title="请补充以下信息以继续分析" />
  <Content>
    <Form layout="vertical">
      <Form.Item label="选择巡检范围" name="scope">
        <Select mode="multiple" options={[
          {value:"core", label:"核心服务"},
          {value:"edge", label:"边缘服务"}
        ]}/>
      </Form.Item>
      <Form.Item label="时间范围" name="timeRange">
        <DatePicker.RangePicker />
      </Form.Item>
    </Form>
  </Content>
  <Footer>
    <Button type="primary">提交</Button>
    <Button>使用默认值</Button>
  </Footer>
</InlineParameterForm>
```

#### 动态表单生成规则
- 智能体在 `task_plans` 中标记一个特殊任务 `require_human_intervention: true`，附带 `params_schema`（JSON Schema 格式），定义需要收集的参数。
- 前端渲染时使用 `react-jsonschema-form` 或 `@rjsf/antd` 自动生成符合深色主题的表单。
- 用户提交后，参数会注入到该任务的 `input_params` 并恢复智能体执行。

### 四、技术实现架构

#### 后端变更
- 扩展 LangGraph 状态，增加 `intervention_type` 字段：`"approval"` / `"form"`。
- `human_interrupt_node` 根据 `intervention_type` 构造不同推送：
  - `approval`：推送安全确认所需的风险信息。
  - `form`：推送 `params_schema`。
- 恢复时处理两种响应：`confirm/decline` 或 `form_data`。

#### WebSocket 事件格式
```json
// 安全确认
{
  "type": "human_intervention",
  "intervention_type": "approval",
  "interrupt_id": "uuid",
  "prompt": "...",
  "risk_level": "high",
  "code_snippet": "...",
  "affected_hosts": ["pay-01"]
}

// 参数收集
{
  "type": "human_intervention",
  "intervention_type": "form",
  "interrupt_id": "uuid",
  "prompt": "请补充巡检参数",
  "schema": { ... }    // JSON Schema
}
```

#### 前端处理
- 在对话消息列表中，拦截 `human_intervention` 事件，不渲染普通气泡，而是渲染对应的 `SecurityConfirmCard` 或 `InlineParameterForm`。
- 用户操作后通过 REST API 回传给后端。

---

### 五、前端组件命名与封装

| 组件名 | 说明 |
|--------|------|
| `SecurityConfirmCard` | 安全确认卡片 |
| `InlineParameterForm` | 内联参数收集表单 |
| `HumanInterventionContainer` | 统一的介入容器，根据 type 渲染子组件 |
| `CodeBlock` | 语法高亮代码块 |
| `RiskTag` | 风险等级标签 |

这些组件放置在 `components/chat/` 目录下，与 `IntentCard`、`PlanCard` 同级。

### 六、与人机协作流程的完整闭环

```
用户提问 → 意图识别 → 任务规划 → 执行...
                                   ↓
                         ┌──需要确认?──安全确认卡片
                         │                ↓用户选择
                         │           确认→继续执行，拒绝→取消任务
                         │
                         └──缺少参数?──参数收集表单
                                         ↓用户提交
                                    注入参数→继续执行
```

这样，所有敏感操作均经过用户明确授权，所有信息缺失均优雅补充，既保证了安全性，又维持了对话的连贯性。
