## AIOpsOS 服务端与客户端配套功能优化

基于“云边一体、管控分离”的设计，我们将服务端与客户端的功能紧密咬合，形成从策略制定、任务编排到现场执行、数据回流的完整闭环。以下从**注册发现**、**配置管控**、**能力授权**、**任务调度**、**数据交互**、**自治协同**、**生命周期管理**、**安全审计**八大维度进行优化。

---

### 一、注册发现与心跳监控

| 功能 | 服务端（Control Plane） | 客户端 Agent |
|------|------------------------|--------------|
| **注册审批** | 接收 Agent 注册请求，根据证书指纹匹配预注册记录；提供审批界面，支持自动批准已信任子网 | 首次启动时携带证书发起注册，提交主机信息、能力清单 |
| **心跳管理** | 维护心跳超时策略（如30s），超时自动标记离线并产生告警事件；提供离线/在线状态统计面板 | 每10s通过WebSocket发送心跳，包含当前负载、磁盘、内存等摘要 |
| **能力上报** | 存储 Agent 上报的能力标签（OS、沙箱类型、可用协议、已装采集器），用于任务调度匹配 | 注册及定期上报能力标签，能力变化时主动推送更新 |

**配套优化点**：引入 **Agent 分组与标签**，服务端可按分组批量管理，客户端自动继承组配置。

---

### 二、配置与模型连接参数下发

| 功能 | 服务端（Control Plane） | 客户端 Agent |
|------|------------------------|--------------|
| **Profile 管理** | 提供可视化 Agent Profile 编辑，支持 YAML/JSON；支持按角色、分组批量下发；所有变更带版本号 | 接收 `config_update` 推送，验签后本地缓存加密存储，热加载生效 |
| **模型连接配置** | 统一管理大模型 API 凭证（Base URL、Key、模型名），按 Agent 角色分配不同模型（如边缘侧使用轻量模型） | 使用下发的 URL+Key 配置本地推理或调用远端模型，启动/停止本地 LLM 进程 |
| **差异化配置** | 支持全局默认值 + 单机覆盖，合并策略清晰 | 本地配置由服务端全量下发，无本地修改能力，保证一致性 |

**配套优化点**：提供 **配置合规检查**，服务端定期扫描在线的 Agent 配置版本，不一致时自动触发同步。

---

### 三、能力授权与角色绑定

| 功能 | 服务端（Control Plane） | 客户端 Agent |
|------|------------------------|--------------|
| **Agent 角色定义** | 定义标准角色（如“Linux生产”、“网络核心”、“存储管理”），每个角色绑定 Skills 白名单、协议权限、资源限制 | 无自主角色决定，仅执行下发的角色和权限约束 |
| **动态授权** | 管理员可在线调整某 Agent 的 Skills 列表或角色，立即推送生效；权限收紧时客户端会拒绝正在执行的任务 | 每次执行 Skill 前检查本地缓存的授权列表；若权限不足，返回明确错误码 |
| **多租户隔离** | 支持将 Agent 分配给不同业务组，结合 RBAC 控制用户可见范围 | 无感知，仅接受服务端调度 |

**配套优化点**：引入 **临时提权审批**，当任务需要超出当前权限的 Skill 时，可发起审批流，管理员确认后临时授权。

---

### 四、任务调度与执行协同

| 功能 | 服务端（Execution Plane） | 客户端 Agent |
|------|--------------------------|--------------|
| **任务派发** | 主智能体规划后生成针对特定 Agent 的执行指令，通过 Execution Plane 经 WebSocket 或 gRPC 下发 | 接收任务，解析为 Skill 调用，放入执行队列（可并行），返回受理确认 |
| **任务优先级与排队** | 支持紧急任务插队，客户端队列满时可拒绝并返回提示 | 内部实现优先级队列，紧急任务立即抢占资源 |
| **异步结果回传** | 接收执行结果，关联到对应会话的 `task_plans` 中，进入合成节点 | 执行完毕后上报详细结果（stdout, stderr, 退出码, 耗时），支持流式输出 |
| **任务取消** | 允许用户手动终止进行中的任务，服务端下发取消指令 | 收到取消信号后，尝试终止运行中的 Skill 进程并回滚已做操作（若支持） |

**配套优化点**：支持 **任务依赖与编排**，服务端可将多个 Agent 的任务编排成工作流（先停服务A → 备份数据 → 启动服务B），客户端仅执行自身环节。

---

### 五、数据采集与事件上报

| 功能 | 服务端（Data Plane） | 客户端 Agent |
|------|---------------------|--------------|
| **指标采集** | 定义采集策略（间隔、指标集），下发到 Agent；接收后存储到时序库，供智能体查询 | 按策略执行采集，本地预聚合后批量推送，避免频繁小包 |
| **日志采集** | 指定日志路径和过滤规则，可动态调整；日志接入 Kafka 供后续分析 | 监听文件或 journald，正则过滤、压缩、流式上传 |
| **事件生成** | 配置本地自治规则，指定何时生成事件；事件经 Kafka 进入告警中心 | 本地判断条件满足时生成 `OpsEvent` 并上送，携带丰富上下文 |
| **数据降噪** | 提供全局重复事件抑制、聚合配置；客户端执行第一层降噪 | 本地短窗口去重，减少无效上报 |

**配套优化点**：服务端提供 **采集策略模板**，一键应用到多台机器。

---

### 六、本地自治与离线容错

| 功能 | 服务端（Control Plane） | 客户端 Agent |
|------|------------------------|--------------|
| **规则下发** | 编写自治规则（如磁盘>90%自动清理日志），配置频率限制和确认策略，下发给 Agent | 加载本地规则引擎，定时评估；执行前若要求确认则暂停等待服务端指令 |
| **离线策略** | 指定离线时的行为：可继续执行自治动作、缓存数据，或仅维持监控 | 断网后切换到离线模式，遵守离线策略，网络恢复后批量同步结果和日志 |
| **冲突处理** | 服务端恢复后接收离线日志，进行审计，冲突时以后续指令为准 | 恢复连接后上报离线期间动作，若与服务端后续指令冲突，以服务端为准回滚 |

**配套优化点**：提供 **离线模拟测试**，在服务端模拟断网场景验证规则合理性。

---

### 七、生命周期管理与自更新

| 功能 | 服务端（Control Plane） | 客户端 Agent |
|------|------------------------|--------------|
| **版本管理** | 管理客户端二进制、沙箱镜像、内置插件版本，提供升级策略（强制/可选） | 启动时上报当前版本，接收升级指令后下载新版本并校验签名 |
| **灰度升级** | 支持按分组或百分比灰度发布，监控升级后健康状态，异常自动回滚旧版本 | 升级前备份当前版本，支持回滚；升级后重新注册并上报新版本 |
| **健康自检** | 接收 Agent 自检报告，异常时自动重启或隔离 | 定期进行健康自检（磁盘、内存、进程），异常时尝试自愈（重启服务），若无法恢复则告警 |
| **远程卸载/禁用** | 管理员可对失陷或退役 Agent 远程禁用（吊销证书）或下发卸载指令 | 收到禁用指令后断开连接并停止所有任务；卸载指令清理自身文件 |

**配套优化点**：客户端支持 **静默升级**，不中断正在执行的长任务（等待任务完成后切换）。

---

### 八、安全审计与合规

| 功能 | 服务端（整个平台） | 客户端 Agent |
|------|-------------------|--------------|
| **通道加密** | 所有通信 mTLS + WSS/gRPC 加密 | 仅接受服务端下发的证书，定期轮换 |
| **操作审计** | 记录每一次从用户→主智能体→客户端→操作系统的完整链路，不可篡改 | 本地记录操作日志，实时或批量上报，本地日志不可删除（只追加） |
| **证书生命周期** | 自建 CA，自动签发和轮换客户端证书；吊销列表 CRL 实时推送 | 证书到期前自动申请续签，更换后无缝重连 |
| **合规检查** | 定期扫描 Agent 配置是否偏离基线，产生合规报告 | 本地配置无修改能力，保证一致性 |

**配套优化点**：引入 **零信任架构**，每次 Agent 执行敏感操作时服务端可要求二次确认，即使 Agent 身份被冒用也难以造成破坏。

---

通过上述八大配套优化，AIOpsOS 的服务端与客户端形成了高度协同的智能运维矩阵，既保证了集中管控的安全与一致性，又赋予边缘侧足够的弹性与自治，全面达成 1-5-10 目标中“10分钟恢复”落地所需的执行闭环。
