# 智能体客户端 (Agent Client)

## 概述

智能体客户端是[[六脉智能运维数智平台研发项目]]中的关键组件，作为[[运维中枢神经大脑]]与各业务系统之间的执行单元，承担数据采集、实时监控和初步分析的任务。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 核心定义

智能体客户端是面向多场景构建的融合式AI智能体，根据业务场景差异化集成对应算法与运维能力。它配合平台服务端实现全链路数据自主采集与分析，实时监控各层级运维对象，自动分析异常、推送告警并生成处置建议。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 关键功能

- **全链路数据采集**：打通运维数据采集“最后一公里”，支持运行态信息收集分析。
- **异常检测与预警**：实时监控系统状态，自动识别异常模式并推送告警。
- **处置建议生成**：结合[[LangChain]]和[[DeepAgents]]框架，基于规则与模型输出处置建议。
- **与平台协同**：配合服务端的多智能体协调机制，执行[[故障取证与根因定界]]等核心任务。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 技术特点

- 基于[[Harness工程]]理念设计，确保智能体行为可控、安全。
- 采用[[DeepAgents]]的子代理生成能力，支持根据不同场景动态创建专业化智能体。
- 与[[智能运维数智平台]]的服务端通过安全栅栏和审计机制交互，降低执行风险。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 在项目中的角色

在“[[运维中枢神经大脑]] + 智能体客户端 + 智能体专家矩阵”混合模式中，智能体客户端作为前哨和数据入口，负责与具体业务系统对接，将采集的信息上传至中枢大脑进行综合推理，并接收中枢的协同指令。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 相关风险与控制

项目团队为智能体客户端设计了安全栅栏和执行审计机制，对于执行类操作增加人工确认环节，确保智能体仅负责数据分析和决策，不直接操控系统。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
