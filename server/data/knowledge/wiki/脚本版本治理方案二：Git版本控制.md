# 脚本版本治理方案二：Git版本控制

方案二利用成熟的Git生态进行脚本版本管理，以ITSM流程驱动变更，并在流程中嵌入AI风险识别。^[raw/脚本版本治理_v18.md]

## 核心思路

- 将所有生产环境脚本统一纳入Git仓库进行版本管理。
- 脚本的新增、修改、删除均需通过ITSM流程发起，并在Git中提交。
- 在ITSM流程中嵌入AI能力，对提交的脚本变更进行自动解读和风险评审。

## 流程描述

1. 脚本开发人员完成脚本开发。
2. 提交ITSM流程（脚本投产服务）。
3. ITSM流程多级审批完成（同时AI自动进行风险评审）。
4. 脚本自动上传至Git仓库。

## 方案优势

- **实施快速**：利用成熟的Git工具，开发量少，能快速搭建并投入使用。
- **技术成熟**：Git作为行业标准工具，稳定性和可靠性有保障。

## 潜在问题

- **管理能力弱**：无法有效控制脚本在服务器上的执行权限、属组属主。
- **管理深度不足**：缺乏运行状态管理，无法实现与服务器脚本的自动比对告警。
