# 六脉项目传统运维痛点分析

## 概述

在[[六脉智能运维数智平台研发项目]]的申报材料中，详细列举了联盟当前运维工作面临的突出痛点，这些痛点是推动[[智能运维数智平台]]建设的核心动因。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 核心痛点

### 人力短缺与效率瓶颈
- **生产系统数量繁杂**：多系统并行运行，传统模式过度依赖人力。
- **人力缺口大**：培养成本持续攀升，响应不及时，处理效率偏低。
- **技术栈多元交叉**：运维人员需掌握多种技术，进一步加剧人力压力。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

### 数据孤岛与故障定界困难
- **运维数据分散**：各业务系统格式杂乱、关联性弱。
- **缺乏高效整合能力**：传统工具分析能力不足。
- **故障“取证”成本高**：根因定界耗时长，严重影响恢复效率。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

### 行业差距与创新迫切性
- 金融机构在AIOps领域已积极布局，联盟仍处于初级验证阶段。
- 需紧跟AI技术迭代趋势，主动应对技术变革带来的挑战。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 前期尝试

生产运维部此前通过对AI故障定界进行了尝试，但因技术手段固化、耗时长等问题，效果受限。这也是转向基于[[DeepAgents]]多智能体架构的重要原因。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
