# 六脉项目AI工程化核心组件

六脉项目（[[六脉智能运维数智平台研发项目]]）在技术选型上集成了当前AI领域的三大核心工程组件：**Harness工程**、**LangChain** 和 **DeepAgents**。这三者构成了项目“运维中枢神经大脑 + 智能体客户端 + 智能体专家矩阵”架构的技术基石。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## Harness工程

**Harness工程**（马具工程）是2026年初由OpenAI正式提出的AI工程化新范式，核心是为AI智能体构建一套完整的运行环境、约束规则与反馈闭环，使AI能够安全、可靠、高效地完成复杂长周期任务。在六脉项目中，Harness工程理念被用于约束智能体行为，确保故障诊断与处置过程的可控性。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

详细定义参见 [[Harness工程]]，其在项目中的具体应用见 [[Harness工程理念在六脉项目中的应用]]。

## LangChain

**LangChain** 是一个开源的LLM应用开发框架，提供大模型接口、提示词管理、记忆、工具调用等模块化能力，充当连接大模型与外部运维系统的桥梁。六脉项目利用LangChain封装各运维工具的API，实现智能体对监控、日志、CMDB等系统的统一调用。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

框架详情参见 [[LangChain]]。

## DeepAgents

**DeepAgents** 是LangChain团队推出的深层智能体框架，基于LangChain和LangGraph构建，支持复杂任务分解、长期规划、子代理协同等深度能力。项目中将DeepAgents作为多智能体协同的核心运行时，负责故障根因推理、处置方案生成以及跨系统的协调调度。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

技术细节参见 [[DeepAgents]]。

## 三者协同关系

在六脉项目中，三者的关系可概括为：
- **Harness工程** 提供工程化约束与安全护栏（参见 [[智能体安全栅栏]]）；
- **LangChain** 提供统一的模型与工具集成框架；
- **DeepAgents** 提供支持长期运行与多智能体协同的应用层能力。

三者共同支撑了项目的[[双向认知沟通模式]]和[[大小模型协同]]机制，目标是构建一套[[智能运维数智平台]]，实现[[故障取证与根因定界]]等核心场景的自动化。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
