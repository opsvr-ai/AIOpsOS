# 六脉项目大脑小脑协同模式

**大脑小脑协同模式**是[[六脉智能运维数智平台]]的核心架构设计，将运维能力类比为生物神经系统中的“大脑”与“小脑”，构建一种双向认知沟通的混合应用模式。该模式旨在突破传统“客户端-服务端”单向运行模式，实现智能运维的主动感知与协同决策。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 核心架构

模式由三大组件构成，各自承担不同职责：

- **[[运维中枢神经大脑]]（大脑）**：位于服务端，承担全局决策、协调调度和逻辑推理职能。大脑基于[[LangChain]]生态的[[DeepAgents]]框架实现，具备多智能体协同能力，负责分析系统状态、生成处置建议、调度专家智能体。
- **[[智能体客户端]]（小脑协同体）**：部署在各业务系统侧，负责数据采集、实时监控、异常分析及预警推送。客户端执行大脑下发的任务，并反馈现场信息，形成闭环。
- **[[智能体专家矩阵]]（专业小脑群）**：由多个运维数字专家智能体组成，每个智能体聚焦特定领域（如数据库、网络、应用），在大脑的统一协调下协作处理复杂故障。

## 工作流程

1. **感知扫描**：智能体客户端7×24小时监控系统运行状态，自动采集全链路数据。
2. **信息上报**：客户端将异常数据、告警信息实时报送至运维中枢神经大脑。
3. **协同研判**：大脑启动专家矩阵，分配子任务到对应的数字专家智能体，各专家独立分析后汇总结果。
4. **决策反馈**：大脑综合各专家意见，进行根因推理和处置建议生成，下发至客户端或操作员。

这种“大脑+小脑”的协同方式，既利用了大脑的宏观统筹能力，又发挥了小脑的专精高效优势，有效提升故障定界和应急响应效率。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 与现有模式的关系

传统运维依赖人工巡检和被动响应，而大脑小脑模式实现了从“人找问题”到“问题找人”的转变。该模式同时也融合了[[大小模型协同]]机制，小模型负责数据提纯，大模型负责逻辑推理，进一步提升整体智能水平。
