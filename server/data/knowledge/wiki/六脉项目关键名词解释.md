# 六脉项目关键名词解释

本文汇总了[[六脉智能运维数智平台研发项目]]中涉及的核心技术名词，取自申报材料的“其他补充说明”部分，供快速查阅。

## Harness工程（Harness Engineering）

Harness工程是2026年初由OpenAI正式提出的AI工程化新范式，核心是为AI智能体构建一套完整的运行环境、约束规则与反馈闭环，让AI安全、可靠、高效地完成复杂长周期任务。

- **类比**：Harness本义是“马具”，工程师设计的是让AI（马）按正确方向奔跑的缰绳和马车。
- **工作重心转移**：从直接编写代码/指令转向设计、构建和迭代AI的运行环境。
- **解决核心痛点**：Agent在大规模落地中的失控、漂移、错误级联、不可持续等问题。
- **核心价值**：通过工程化手段实现对AI行为的约束、校验、执行、记忆、安全管控与错误自愈，让AI进化成能自主交付项目的工程师。

详细参见：[[Harness工程]] 及 [[Harness工程理念在六脉项目中的应用]]。

## LangChain

LangChain是一个开源框架（Python/JS双版本），专门用于构建大语言模型（LLM）应用程序，核心定位是“连接大模型与外部资源/工具的桥梁”。

- **模块化设计**：将LLM能力与外部资源、业务逻辑进行有机整合。
- **统一接口**：为不同LLM提供商（如OpenAI、通义千问等）提供标准化接口。
- **核心模块**：Models、Prompts、Chains、Memory、Tools、Agents、Retrieval（RAG技术）。
- **应用场景**：企业知识库问答、AI数据分析助手、AI自动化工作流、智能客服等。

详见：[[LangChain]] 及相关应用页面。

## DeepAgents

DeepAgents是LangChain团队推出的AI代理框架，基于LangChain和LangGraph构建，专门用于开发能够执行复杂、开放式任务并在较长时间跨度内持续运行的“深层智能体”。

- **分层架构**：在LangChain（框架层）和LangGraph（运行时层）基础上构建应用层。
- **核心能力**：自主规划、文件系统访问、子代理生成、对话压缩、结果回收。
- **技术定位**：LangChain提供核心抽象，LangGraph聚焦运行时稳定性，DeepAgents提供开箱即用的应用层工具集。

参见 [[DeepAgents]] 以及 [[六脉项目AI工程化核心组件]]。

## 三者关系

- [[Harness工程]] 是顶层理念，指导如何工程化AI行为。
- [[LangChain]] 提供了连接LLM与外部世界的脚手架。
- [[DeepAgents]] 则在LangChain之上专门面向复杂长周期任务提供应用层实现。
- 三者共同构成了[[六脉智能运维数智平台]]的AI工程基石。
