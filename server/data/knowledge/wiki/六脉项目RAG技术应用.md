# 六脉项目RAG技术应用

**RAG（Retrieval-Augmented Generation，检索增强生成）**是[[LangChain]]框架的核心模块之一，在[[六脉智能运维数智平台研发项目]]中被用于解决运维知识碎片化和故障信息检索低效的问题。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 应用背景

联盟运维工作中，生产系统数量繁多、技术栈多元，故障发生后“取证”与分析成本居高不下。传统运维工具缺乏高效的数据整合与检索能力，导致故障定界耗时长。RAG技术能够将历史故障案例、运维手册、系统日志等非结构化知识向量化并存储，在需要时快速检索出最相关的上下文供大模型参考，从而提升推理准确性。

## 在六脉项目中的具体实现

- **知识库构建**：将历史故障记录、应急处置流程、系统配置文档等运维知识转化为向量索引，存储于知识库中。
- **检索阶段**：当智能体（如[[运维数字专家智能体]]）遇到异常事件时，自动从知识库中检索相似案例或对应处置步骤。
- **增强生成**：将检索到的上下文注入大模型提示词，辅助模型进行根因分析和建议生成，减少幻觉。

RAG技术尤其适用于[[故障取证与根因定界]]场景，使得大小模型协同机制中的大模型能够基于事实知识而非仅凭参数记忆进行推理。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 与其他模块的关系

- **[[LangChain]]**：提供统一的Retrieval模块接口，支持多种向量数据库和检索策略。
- **[[DeepAgents]]**：在智能体规划任务时，可主动调用RAG模块获取必要信息。
- ****：客户端采集的数据经过预处理后也可动态纳入检索范围，实现实时知识更新。

未来随着平台迭代，将逐步扩展RAG的应用范围至变更影响分析、容量规划等更多运维场景。
