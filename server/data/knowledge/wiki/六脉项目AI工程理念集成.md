# 六脉项目AI工程理念集成

[[六脉智能运维数智平台研发项目]]在AI工程化路径上采用了前沿理念组合：以 **Harness工程** 为指导思想，依托 **LangChain** 生态的 **DeepAgents** 框架实现多智能体协同，构建“运维中枢神经大脑 + 智能体客户端 + 智能体专家矩阵”的混合架构。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 核心理念衔接

| 理念/工具 | 在六脉项目中的角色 | 关系 |
|-----------|-------------------|------|
| [[Harness工程]] | 顶层设计哲学，定义AI智能体的运行环境、约束规则与反馈闭环，确保智能体行为安全可控 | 指导LangChain生态的工程化落地，解决失控、漂移等问题 |
| [[LangChain]] | 框架层，提供大模型统一接口、提示词管理、工具调用等模块化能力 | 作为DeepAgents的底层基础设施，连接大模型与外部运维工具 |
| [[DeepAgents]] | 应用层，提供开箱即用的子代理生成、自主规划、文件访问等深度能力 | 直接用于构建运维数字专家智能体的自主决策与协同 |
| [[运维中枢神经大脑]] | 系统架构层面的中心调度与决策节点 | 基于DeepAgents实现多智能体协同调度 |
| [[智能体专家矩阵]] | 面向不同运维场景的专业智能体集合 | 由DeepAgents子代理生成机制动态创建 |

## 双向认知沟通模式

项目突破传统“客户端-服务端”单向模式，提出“大脑+小脑”场景：
- **大脑**：运维中枢神经大脑（服务端）进行全局推理、任务分解与决策协调；
- **小脑**：各个运维数字专家智能体（客户端）执行具体监测、数据采集与局部决策。

这一模式得益于Harness工程对智能体行为的严格约束与LangChain/DeepAgents对复杂任务的拆解能力。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 技术路线选择依据

项目技术预研表明，早期Dify工作流存在执行模型固化、耗时长等问题，而基于LangChain生态的DeepAgents路线可有效解决这些弊端。同时，项目优先选用头部大厂使用的技术路线（如OpenAI、LangChain），保障技术稳定性和可持续性。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 相关风险与应对

- **不可控安全风险**：通过[[智能体安全栅栏]]、执行审计验证与人工确认机制，确保智能体仅负责数据分析与决策，不直接执行操作。
- **技术迭代风险**：持续跟踪Harness工程和DeepAgents的最新版本，保持架构适配性。
- **算力不足**：当前联盟算力规划无法满足满血运行，计划先选择试点系统进行路径验证。
