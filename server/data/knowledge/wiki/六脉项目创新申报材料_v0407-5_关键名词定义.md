# 六脉项目创新申报材料 v0407-5 关键名词定义

本文档提取了申报材料中“其他补充说明”部分的三个关键名词解释，并关联到相关页面。

## Harness工程（Harness Engineering）

**核心定义**：2026年初由OpenAI提出的AI工程化新范式，为AI智能体构建完整的运行环境、约束规则与反馈闭环，使其安全可靠地完成复杂长周期任务。

**关键特点**：
- 喻为“马具”，工程师设计让AI按正确方向奔跑的缰绳和马车。
- 工作重心从编写代码转向设计AI运行环境。
- 解决Agent失控、漂移、错误级联等问题。

**核心价值**：通过工程化手段约束、校验、执行、记忆、安全管控与错误自愈，使AI从“聊天机器人”进化为“能自主交付项目的工程师”。

详见独立页面 [[Harness工程]] 及应用案例 [[Harness工程理念在六脉项目中的应用]]。

## LangChain

**核心定义**：开源框架，专门构建大语言模型（LLM）应用，核心定位为“连接大模型与外部资源/工具的桥梁”。

**关键特点**：
- 模块化设计，统一接口，突破LLM上下文窗口限制。
- 核心模块包括Models、Prompts、Chains、Memory、Tools、Agents、Retrieval（RAG）。

**应用场景**：企业知识库问答、AI数据分析助手、自动化工作流、智能客服等。

详见独立页面 [[LangChain]] 及在项目中的应用 [[六脉项目AI工程化核心组件]]。

## DeepAgents

**核心定义**：LangChain团队推出的AI代理框架，基于LangChain与LangGraph构建，开发能够执行复杂、开放式任务并在较长时间跨度内持续运行的“深层智能体”。

**关键特点**：
- 分层架构（框架层+运行时层+应用层）。
- 深度能力：规划工具、文件系统后端、子代理生成、对话压缩、结果回收。
- 解决传统LLM工具调用代理的“肤浅”问题。

**技术定位**：LangChain提供框架抽象，LangGraph聚焦运行时稳定性，DeepAgents提供开箱即用的应用层工具集。

详见独立页面 [[DeepAgents]] 及与项目相关的 [[六脉项目AI工程化范式演进]]。
