# 六脉项目故障处置新方法

本项目针对运维中最突出的**故障根因定界慢、定位难**痛点，提出了基于AI多智能体协同的混合处置方法，结合大模型与小模型各自优势，形成全新的故障处置范式。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 核心方法：大模型+小模型协同

- **小模型**：负责算法提纯，快速筛选、提炼故障关键数据，减少噪音干扰。
- **大模型**：侧重逻辑推理，辅助研判故障根因，快速给出处置建议。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

该模式突破了以往单一大模型工作流固化、耗时长的问题，是[[六脉项目故障定界新方法]]的深化实践。

## 多智能体协同机制

- 构建“运维中枢神经大脑 + 智能体客户端 + 智能体专家矩阵”协同架构，将故障处置从单人排查升级为多智能体协作。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
- 各[[运维数字专家智能体]]在统一调度下实现信息共享与任务分解，解决应急过程中多岗位信息协同不畅的问题。
- 通过[[智能体安全栅栏]]确保只分析决策、不直接执行操作，保障安全可控。

## 应用场景

- **故障应急处置**：自动采集全链路数据，分析异常并推送告警与处置建议，提升应急协同效率。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
- **日常运维**：7×24小时系统监控与状态分析，实现隐患预测与辅助处置前置。

## 与已有方法的差异

相较于去年通过Dify工作流进行的AI故障定界尝试，本方法采用[[DeepAgents]]框架，支持长期运行、自主规划与子代理生成，克服了执行模型固化、耗时长等弊端。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
