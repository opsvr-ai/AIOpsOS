# 六脉项目创新申报摘要

本文摘要了生产运维部申报的 **【六脉】智能运维数智平台研发项目** 的核心内容，该申报材料详细描述了项目的背景、目标、方案、风险与计划。更多细节可参阅各专题页面。

## 项目概览

- **创新名称**：[[六脉智能运维数智平台研发项目]]
- **申报部门**：生产运维部
- **创新类型**：内部创新
- **采购情况**：采购，预计金额100万元

以上信息详见 [[六脉项目基本信息]] 及 [[六脉项目采购与服务]]。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 工作背景

联盟当前面临运维人力紧张、系统繁杂、数据分散、故障定界困难等痛点（详见 [[六脉项目行业趋势与联盟运维挑战]]），同时行业头部已加速AIOps布局。去年通过 [[六脉项目前期Dify工作流验证]] 进行了初步尝试，但受技术限制效果有限。今年基于 [[OpenClaw]] 技术和 [[DeepAgents]] 路线探索新方案。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 工作目标

详见 [[六脉项目工作目标]]。

1. **构建多智能体协同核心架构**：打造 [[运维中枢神经大脑]] + [[agent-client]] + [[智能体专家矩阵]] 体系。
2. **聚焦故障应急场景**：采用 [[大小模型协同]] 提升 [[故障取证与根因定界]] 效率，融入 [[fault-emergency-agent-collaboration]]。
3. **立足路径实现验证**：推进与 [[21+6关键业务系统]] 等场景深度融合。

## 主要工作内容

详见 [[六脉项目主要工作内容]] 及 [[六脉项目智能体协同架构]]。

1. **智能运维数智平台**：构建多智能体协同平台，集成 [[Skills动态发现]] 等能力。
2. **智能体客户端**：面向多场景的 [[六脉项目智能体客户端功能]]，实现数据自主采集与分析。
3. **高质量运维场景建设**：聚焦 [[六脉项目故障定界与应急协同解决方案]] 及 [[六脉项目高质量运维场景建设]]。

## 工作风险分析

详见 [[六脉项目工作风险分析]]、[[六脉项目问题与对策]]。

- **智能体安全风险**：通过 [[智能体安全栅栏]] 及人工确认应对。
- **技术迭代风险**：采用主流技术路线，保障稳定性。
- **人力不足风险**：采用联盟主导+外部辅助模式。
- **算力不足问题**：需试点先行，详见 [[六脉项目算力不足问题与对策]]。
