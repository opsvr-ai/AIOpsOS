# 六脉项目工作目标

六脉项目（[[六脉智能运维数智平台研发项目]]）旨在以AI多智能体架构为核心，依托LangChain生态[[DeepAgents]]多智能体框架，探索“**[[运维中枢神经大脑]] + + [[智能体专家矩阵]]**”混合应用模式，复用现有运维工具能力搭建核心框架，分阶段构建主动、智能、协同、安全的一体化运维体系。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

鉴于当时AI+运维领域尚处于探索攻坚阶段，本项目立足联盟特色，以构建安全可靠、符合运维需求的AI运维平台为核心目标，聚焦运维工作中**[[故障取证与根因定界]]**难等核心痛点，系统性探索解决方案，并为后续智能化运维转型沉淀能力。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 多智能体协同核心架构

平台将打造以多智能体协作为核心的全新架构体系，支持联盟根据自身运维实际需求，自主构建不同角色、明确职责边界的运维数字专家智能体，形成专业化、体系化的**[[多智能体运维数字专家体系]]**。各运维数字专家智能体在统一协调调度下，实现与联盟现有各类工具平台的深度对接与高效联动。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 聚焦故障应急场景，提升处置效率

针对故障根因定界慢、定位难的核心痛点，深度融合AI多智能体协同技术，搭建**[[大小模型协同]]**联动模式：小模型负责算法提纯，快速筛选故障关键数据；大模型侧重逻辑推理，辅助研判故障根因并给出处置建议。同时尝试将运维数字专家智能体融入应急处置流程，重点解决应急过程中多岗位信息协同不畅的问题，提升应急协同效率。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 立足路径实现验证，推进场景深度融合

聚焦AI运维平台落地路径的可行性验证，在夯实路径落地成效的基础上，持续推进平台与联盟各类运维场景的深度融合，通过迭代优化平台功能、完善技术架构，最终建成功能完备、高效协同、智能可控的智能化运维平台，推动AI+运维场景不断深化。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
