# 六脉项目安全风险与算力约束

## 概述

六脉项目（[[六脉智能运维数智平台研发项目]]）在创新申报中识别了多项风险与约束，包括智能体运行安全、技术迭代、人力储备和算力不足等。这些因素直接影响项目的可行性与实施策略。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 风险与应对

### 1. 智能体运行不可控安全风险
- **描述**：AI智能体在执行过程中可能出现失控或错误级联。
- **应对**：
  - 在工程实现环节增加 **[[智能体安全栅栏]]** 及执行审计验证。
  - 对执行类操作增加人工确认环节。
  - 在场景探索中，智能体仅负责数据分析与决策，不直接执行操作。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

### 2. 技术更新迭代带来开发落地风险
- **描述**：AI技术发展迅速，可能导致选型路径过时或稳定性不足。
- **应对**：优先选用头部厂商使用的技术路线（如[[Harness工程]]、[[DeepAgents]]），保障技术路线的稳定性和可持续性。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

### 3. 技术储备及人力投入不足风险
- **描述**：新技术引入面临内部团队经验不足、相关领域人才缺失。
- **应对**：采用 **以联盟人员为主，外部咨询服务和研发力量为辅** 的合作模式，确保自主可控与技术延续。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

### 4. 算力不足问题
- **描述**：多智能体协同模式对联盟算力供给能力和并发处理能力要求极高，当前算力规划无法满足满血运行。
- **应对**：先选择试点系统进行路径探索和能力验证，逐步积累经验。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 对项目计划的影响

这些风险与约束已反映在项目计划中（参见[[六脉项目工作计划-2026]]），基础版上线（2026年8月）仅覆盖服务端和客户端以及多智能体协作能力，后续阶段根据验证结果逐步扩展。算力约束是平台推广前必须克服的关键瓶颈。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
