# 运维研发脚本管理现状数据（2026年2月）

该页面汇总了[[脚本版本治理_v18]]中2026年2月5日至9日期间运维及研发提供的脚本管理现状调查数据。

## 整体概况

- **涉及关键业务系统**：13 个
- **收集脚本总数**：141 个
- **运维编写**：82 个 (58%)
- **研发提供**：59 个 (42%)

| --- | --- |

| 应急处置类 | 21% |

| 其他 | 21% |

## 版本管理情况

- **无版本管理**：65%
- **有版本管理（代码库）**：35%

## 生产环境执行方式

- **生产服务器手工执行**：70%
- **自动化/其他方式**：30%

## 发现的主要问题

- 多数脚本缺乏版本控制，散落于个人终端或共享目录，存在[[脚本版本治理问题分析|版本管理缺失]]风险。
- 生产环境大量依赖手工执行，未接入[[六脉智能运维数智平台|自动化平台]]调度。
- 仅35%的脚本纳入了代码库版本管理，反映治理紧迫性。
