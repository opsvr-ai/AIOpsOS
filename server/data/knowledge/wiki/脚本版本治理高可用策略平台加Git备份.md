# 脚本版本治理高可用策略（平台+Git备份）

该策略是[[脚本版本治理方案推荐（混合模式）]]的核心组成部分，旨在通过平台集中管理与Git分布式存储相结合，实现脚本版本治理的高可用性与业务连续性。

## 设计背景

[[脚本版本治理方案比较与推荐_2026_03]]中指出，纯平台集中管理模式存在单点故障风险，而纯Git版本控制缺乏精细化的权限与告警能力。混合模式取长补短，引入Git作为容错备份。

## 核心机制

- **双模式管理**：平台同时提供“托管分发”与“原生执行”两种模式，兼顾灵活性与可控性。
- **自动同步备份**：所有平台托管的脚本自动同步至Git仓库，平台故障时可快速从Git恢复。
- **一致性与告警**：定时扫描服务器脚本与平台记录，不一致时触发告警，防止未授权篡改。

## 优势

- **双重保障**：平台故障时Git仍可提供脚本历史版本，保障灾备切换、批量执行等核心运维场景。
- **精细化管理**：保留平台对脚本权限（属主/属组）、部署路径、分发等控制。
- **合规追溯**：Git完整历史与平台版本记录互相印证，满足审计要求。

## 适用场景

该策略特别适用于对业务连续性要求高的金融科技公司，如[[六脉项目赋能联盟成员行生态]]中的灾难恢复与变更管理场景。
