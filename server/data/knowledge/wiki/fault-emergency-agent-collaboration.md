# 故障应急智能体协同 (Fault Emergency Agent Collaboration)

## 定义

故障应急智能体协同是[[六脉智能运维数智平台研发项目]]中核心应用场景之一，旨在利用多智能体技术提升运维故障处置效率，重点解决根因定界慢、定位难以及应急协同不畅等问题。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 核心思路

采用“大模型+小模型”协同联动模式：
- **小模型**（算法模型）负责数据提纯和快速筛选，从海量运维数据中提取关键特征。
- **大模型**（LLM）侧重逻辑推理，辅助研判故障根因并给出处置建议。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

该模式将[[智能体客户端]]采集的数据汇聚至[[运维中枢神经大脑]]，再由[[多智能体运维数字专家体系]]中的不同专家智能体分工协作，实现快速定界与响应。

## 关键环节

1. **故障取证与根因定界**：通过多智能体协同自动收集相关日志、指标、告警，结合[[LangChain]]的检索增强生成（RAG）能力关联历史故障知识，快速定位根因。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
2. **应急协同**：将运维数字专家智能体融入应急处置流程，自动召集相关岗位智能体（如网络、数据库、应用），共享信息并协调处置步骤，解决多岗位信息不对称问题。
3. **处置建议生成**：基于[[DeepAgents]]的自主规划能力，分解复杂故障处理步骤，生成可人工确认的行动方案。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 技术基础

- [[Harness工程]]：为智能体行为提供约束与安全反馈闭环，确保自主决策可靠。
- [[DeepAgents]]：提供长期规划、子代理生成、记忆管理等能力，支撑复杂故障处理流程。
- [[LangChain]]：统一框架实现工具调用、Prompt管理、外部知识检索。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 预期价值

- 将故障定界时间从小时级缩短至分钟级，大幅降低MTTR。
- 减少应急中对人工经验的高度依赖，提升协同效率。
- 为[[智能运维数智平台]]沉淀故障处理知识库，持续优化模型。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 风险与应对

由于智能体参与决策可能带来不可控风险，项目采用安全栅栏、执行审计以及人工确认机制，确保智能体仅辅助分析，不直接操控系统。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
