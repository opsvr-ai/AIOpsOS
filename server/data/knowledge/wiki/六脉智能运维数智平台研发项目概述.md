# 六脉智能运维数智平台研发项目概述

**六脉智能运维数智平台研发项目**是生产运维部申报的2026年公司级内部创新项目，旨在以AI多智能体架构为核心，构建“运维中枢神经大脑 + 智能体客户端 + 智能体专家矩阵”混合应用模式，重点解决传统运维中故障定界难、应急协同慢等痛点。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 基本信息
- **创新名称**：【六脉】智能运维数智平台研发项目
- **申报部门**：生产运维部
- **申报类型**：新增
- **创新类型**：内部创新
- **服务采购**：采购，预计金额100万元
- **新技术**：基于[[Harness工程]]理念，依托[[LangChain]]生态[[DeepAgents]]框架
- **新方法**：突破传统“客户端-服务端”单向模式，构建“运维中枢神经大脑+智能体客户端+智能体专家矩阵”[[双向认知沟通模式]]，探索“大脑+小脑”协同模式
- **新思路**：为[[21+6关键业务系统]]定制专属[[运维数字专家智能体]]，实现7×24小时监控与在线协作

## 核心目标
1. 构建[[多智能体运维数字专家体系]]核心架构，实现智能体协同与工具有效联动
2. 聚焦故障应急场景，通过[[大小模型协同]]提升根因定界与处置效率
3. 验证落地路径，推进与各类运维场景深度融合

## 主要工作内容
- **智能运维数智平台**：围绕“大脑+智能体客户端+专家矩阵”构建多智能体协同平台，集成[[Skills动态发现]]等能力
- **智能体客户端**：面向多场景的融合式AI客户端（[[agent-client]]），实现全链路数据采集与异常分析
- **高质量运维场景建设**：聚焦[[故障取证与根因定界]]、应急协同核心需求（[[fault-emergency-agent-collaboration]]）

## 风险与应对
项目识别了[[六脉项目安全风险与算力约束]]、技术迭代风险、人力储备风险及算力不足风险，并制定了相应策略（参见[[六脉项目合作与自主可控策略]]）。
