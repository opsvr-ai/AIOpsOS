# 六脉项目创新申报材料-风险分析与应对措施

本页面提炼自《公司级创新申报材料-生产运维部_v0407-5》中“工作风险分析”部分，系统归纳了项目面临的四大风险及相应应对策略，并链接至各专项讨论页面。

## 风险一：智能体运行不可控的安全风险

**核心描述**：多智能体自主运行可能产生不可预知的行为，存在失控、错误级联等安全隐患。

**应对措施**：
- 在工程实现环节增加安全栅栏与执行审计验证；
- 对执行类操作增加人工确认环节；
- 智能体仅负责数据分析与决策，不直接操作执行。

详细安全设计参见 [[智能体安全栅栏]] 与 [[六脉项目安全风险与算力约束]]。

## 风险二：技术更新迭代带来的项目开发落地风险

**核心描述**：AI技术迭代迅速，所选技术栈可能快速过时，导致项目开发路径失效。

**应对措施**：
- 优先选用头部大厂（如OpenAI、LangChain）使用的成熟技术路线，保障技术稳定性。

相关技术选型依据参见 [[六脉项目创新申报技术路线选型依据]] 与 [[Harness工程理念在六脉项目中的应用]]。

## 风险三：技术储备及人力投入不足风险

**核心描述**：新技术的应用需相应技术人才，内部储备不足可能导致项目进度滞后或依赖单一供应商。

**应对措施**：
- 采用以联盟人员为主、采购外部咨询服务与研发力量为辅的合作模式；
- 确保自主可控与技术的可持续演进。

详见 [[六脉项目合作与自主可控策略]] 与 [[六脉项目采购与服务]]。

## 风险四：算力不足问题

**核心描述**：多智能体协同及大模型推理对算力要求极高，近期联盟算力规划无法支撑满血运行。

**应对措施**：
- 先选取试点系统进行路径探索与能力验证，逐步扩展。

算力约束的详细分析参见 [[六脉项目算力不足应对方案]] 与 [[六脉项目安全风险与算力约束]]。

## 综合风险应对策略

项目文档还强调了“以联盟人员为主”的自主可控策略，以及通过分阶段验证降低技术风险的整体思路。全面的风险应对框架参见 [[六脉项目风险应对综合策略]]。
