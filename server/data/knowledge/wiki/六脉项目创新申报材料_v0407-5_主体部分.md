# 六脉项目创新申报材料 v0407-5 主体部分

## 基本信息

- **创新名称**：[[六脉智能运维数智平台研发项目]]
- **申报部门**：生产运维部
- **负责人**：待补充

- **服务采购**：采购（预计采购金额100万元）

## 工作背景

联盟当前运维面临生产系统繁杂、人力不足、数据分散且故障定界难等痛点。行业头部厂商已加速AIOps布局，联盟尚处初级阶段。详见 [[六脉项目工作背景]] 和 [[六脉项目传统运维痛点分析]]。

## 工作目标

构建以[[多智能体运维数字专家体系]]为核心的智能运维平台，聚焦故障应急场景提升处置效率，并验证落地路径。分为三个子目标：

1. 构建多智能体协同核心架构 → [[六脉项目工作目标-构建多智能体协同核心架构]]
2. 聚焦故障应急场景 → [[六脉项目工作目标-聚焦故障应急场景]]
3. 立足路径实现验证 → [[六脉项目工作目标-立足路径实现验证]]

详细信息参见 [[六脉项目工作目标]]。

## 主要工作内容

项目围绕“运维中枢神经大脑+智能体客户端+智能体专家矩阵”构建三大模块：

1. **智能运维数智平台**：服务端集成多智能体协同、Skills动态发现等能力，实现自主扫描、隐患预测。→ [[六脉项目主要工作内容-智能运维数智平台详细]]，框架设计参见 [[智能运维数智平台]]。
2. **智能体客户端**：面向多场景的AI客户端，实现全链路数据采集与异常分析。→ [[六脉项目主要工作内容-智能体客户端详细]]，架构细节见 [[agent-client]]。
3. **高质量运维场景建设**：聚焦故障取证与根因定界、应急协同。→ [[六脉项目主要工作内容-高质量运维场景建设]]，核心能力见 [[故障取证与根因定界]]。

## 工作风险分析

项目面临四大风险及应对策略：

1. **智能体安全风险**：设置安全栅栏，人工确认执行，智能体不直接操作。详细见 [[智能体安全栅栏]] 及 [[六脉项目智能体安全风险应对]]。
2. **技术更新迭代风险**：选用头部大厂技术路线。参见 [[六脉项目创新申报技术路线选型依据]]。
3. **技术储备与人力不足**：采用联盟人员为主、外部咨询为辅的合作模式。参见 [[六脉项目合作与自主可控策略]]。
4. **算力不足**：先试点系统，路径验证。参见 [[六脉项目算力不足问题与对策]]。

综合应对策略见 [[六脉项目风险应对综合策略]]。
