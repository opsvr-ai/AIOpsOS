# 六脉项目名词解释体系

在[[六脉智能运维数智平台研发项目]]的申报材料中，系统阐述了支撑平台落地的三大核心技术概念：[[Harness工程]]、[[LangChain]]和[[DeepAgents]]。这三者构成了项目的AI工程理念基座（参见[[六脉项目AI工程理念集成]]）。

## 三大名词的关系

- **Harness工程**：作为AI工程化的新范式，为整个平台提供“马具”式的约束、校验与反馈闭环，保证大模型在运维场景下安全、可控地执行任务 ^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
- **LangChain**：作为连接大模型与外部工具的开源框架，为平台提供模块化的“桥梁”能力，统一接口管理模型、提示词、记忆、工具调用等 ^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
- **DeepAgents**：作为LangChain生态的应用层框架，提供开箱即用的“深层智能体”开发工具，支持任务分解、文件操作、子代理生成等复杂长周期运维场景 ^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 在项目中的定位

项目采用“运维中枢神经大脑+智能体客户端+智能体专家矩阵”的混合模式（见[[多智能体运维数字专家体系]]），其中：

- Harness工程理念指导整体工程架构设计，确保智能体行为可解释、可审计
- LangChain提供底层框架整合能力，对接联盟现有运维工具与外部LLM
- DeepAgents用于构建具体的运维数字专家智能体，实现故障定界、应急协同等场景（参见[[六脉项目故障定界新方法]]）

## 技术选型优势

该体系解决了传统AI工作流（如Dify）执行模型固化、耗时长的问题，通过动态技能发现（[[Skills动态发现]]）和智能体协同，实现更灵活的故障数据处理和推理决策（参见[[大小模型协同]]）。
