# 六脉项目新技术方法思路

## 概述

[[六脉智能运维数智平台研发项目]]在技术路线上引入了2026年AI工程化前沿理念，融合Harness工程、LangChain生态及DeepAgents多智能体框架，并创新性地提出“运维中枢神经大脑+智能体客户端+智能体专家矩阵”双向认知沟通模式，旨在突破传统运维单向运行瓶颈。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 新技术运用

- **Harness工程**：参考OpenAI提出的AI工程化范式，为智能体构建运行环境、约束规则与反馈闭环，确保AI行为可靠可控。详见[[Harness工程]]。
- **LangChain生态**：作为连接大模型与外部资源的桥梁，提供统一接口和模块化能力，支撑智能体与工具平台的深度对接。详见[[LangChain]]。
- **DeepAgents框架**：基于LangChain和LangGraph构建的深层智能体框架，支持长期规划、子代理生成及自主迭代，用于实现复杂运维任务。详见[[DeepAgents]]。

## 新方法创新

打破传统“客户端-服务端”单向模式，构建“**运维中枢神经大脑 + 智能体客户端 + 智能体专家矩阵**”双向认知沟通体系：^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
- **运维中枢神经大脑**：作为决策与数据中枢，集中处理多源信息，分发任务。详见[[运维中枢神经大脑]]。
- **智能体客户端**：部署于业务系统侧，实现全链路数据自主采集与分析，实时监控并推送告警。详见[[agent-client]]。
- **智能体专家矩阵**：为关键业务系统定制专属数字专家智能体，构成多智能体协同网络。详见[[智能体专家矩阵]]。

## 新思路探索

计划为联盟21+6关键业务系统定制[[运维数字专家智能体]]，通过7×24小时监控与状态分析，推动中枢大脑与各业务系统数字专家**在线协作**，构建AI数字化运维专家群，提升故障响应与处置效率。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 与现有体系的集成

- 复用现有运维工具能力（如[[脚本版本治理_v18]]等），逐步替换传统人工依赖。
- 依托[[大小模型协同]]机制，小模型负责算法提纯，大模型侧重逻辑推理，加速故障定界。

## 意义与目标

该项目标志着联盟从传统运维向AI智能运维转型的关键一步，旨在积累AIOps实操经验，为后续全面智能化奠定技术基础。详见[[六脉项目工作目标]]。
