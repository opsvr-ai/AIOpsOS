# 六脉项目AI工程化范式演进

六脉项目在设计理念上深度借鉴了当前AI领域最前沿的工程化范式，从传统的人工编码运维向AI智能体工程化演进。项目的技术栈选择与架构设计反映了从“工具层 → 框架层 → 应用层”的完整范式升级。

## 范式演进的三层结构

### 1. 传统工具层：LangChain框架
[[LangChain]] 作为开源框架，承担着连接大语言模型与外部资源/工具的桥梁作用^[raw/公司级创新申报材料-生产运维部_v0407-5.md]。在六脉项目中，LangChain的Modules（Models、Prompts、Chains、Memory、Tools、Agents、Retrieval）为平台提供了基础的模块化能力，是构建智能体应用的基础设施。

### 2. 运行时层：LangGraph与DeepAgents
[[DeepAgents]] 建立在LangChain和LangGraph之上，专注于解决传统LLM工具调用的“肤浅”问题，支持长期规划、子代理生成、对话压缩等深度能力^[raw/公司级创新申报材料-生产运维部_v0407-5.md]。它使得智能体能够处理复杂、开放式的长周期运维任务。

### 3. 工程化层：Harness工程理念
[[Harness工程]] 是OpenAI提出的AI工程化新范式，核心是为AI智能体构建完整的运行环境、约束规则与反馈闭环^[raw/公司级创新申报材料-生产运维部_v0407-5.md]。六脉项目将其落地为“运维中枢神经大脑+智能体客户端+智能体专家矩阵”的混合模式，实现了从“只会聊天的机器人”到“能自主交付任务的运维数字专家”的进化。

## 六脉项目中的范式落地

六脉项目通过以下方式实践AI工程化范式演进：
- **安全约束**：设置[[智能体安全栅栏]]，对执行级操作增加人工确认环节^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
- **协同机制**：采用[[运维中枢神经大脑]]统一调度[[智能体专家矩阵]]，形成[[双向认知沟通模式]]
- **能力复用**：基于[[Skills动态发现]]机制，实现工具能力的动态注册与调用

该范式演进使得六脉项目能够突破传统“客户端-服务端”单向模式，构建可迭代、可扩展的智能运维体系。

## 核心价值

范式升级带来的核心价值在于：工程师的工作重心从编写具体指令转向设计AI的运行环境与约束规则，每当AI犯错时就通过工程化手段固化规避方案，最终实现运维能力的持续进化与自主可控。这与六脉项目“以联盟人员为主采购外部服务为辅”的[[六脉项目合作与自主可控策略]]相一致。
