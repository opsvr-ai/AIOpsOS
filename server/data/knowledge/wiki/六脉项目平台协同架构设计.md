# 六脉项目平台协同架构设计

六脉智能运维数智平台的核心架构采用“**运维中枢神经大脑 + 智能体客户端 + 智能体专家矩阵**”的混合模式，旨在打破传统“客户端-服务端”单向运行模式，构建双向认知沟通体系，实现“大脑+小脑”的协同运维场景。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 架构组件

- **运维中枢神经大脑**：作为平台决策核心，负责多智能体协同调度、Skills动态发现、生产系统运行自主扫描、隐患预测及辅助处置前置。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
- **智能体客户端**：面向多场景的融合式AI客户端，集成差异化算法与运维能力，实现全链路数据自主采集与分析，实时监控各层级运维对象，自动分析异常并推送告警与处置建议。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
- **智能体专家矩阵**：由多个专业化运维数字专家智能体组成，各智能体职责边界明确，在统一协调调度下与现有工具平台深度对接、高效联动。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 协同模式

- **双向认知沟通**：中枢大脑与客户端、专家矩阵之间不再是简单的命令-响应，而是持续的双向认知交互，大脑下发策略，客户端与专家反馈实时状态与处理结果。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
- **大小模型联动**：小模型负责算法提纯、快速筛选故障关键数据，大模型侧重逻辑推理、辅助根因研判，提升故障处置效率。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
- **安全护栏**：智能体运行中增加安全栅栏及执行审计验证，执行类操作需人工确认，智能体仅负责数据分析与决策，不直接执行操作。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
