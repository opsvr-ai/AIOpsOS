# 六脉智能运维数智平台

六脉智能运维数智平台是生产运维部申报的公司级创新项目，旨在利用AI多智能体技术重构传统运维模式，解决人力不足、效率偏低、被动响应等痛点。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 概述

该平台以“**运维中枢神经大脑 + 智能体客户端 + 智能体专家矩阵**”为核心理念，基于[[Harness工程]]理念和[[LangChain]]生态的[[DeepAgents]]框架进行功能实现。平台尝试突破传统“客户端-服务端”单向运行模式，构建双向认知沟通模式，探索“大脑+小脑”运维场景。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 工作背景与目标

当前运维面临生产系统数量繁杂、技术栈多元、数据分散等突出痛点，传统运维过度依赖人力，故障定界困难。联盟在AIOps领域尚处初级阶段，亟需补齐技术差距。本项目以构建安全可靠、符合运维需求的AI运维平台为核心目标，聚焦故障根因定界慢、应急协同不畅等问题，分阶段建设主动、智能、协同、安全的一体化运维体系。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 核心架构与工作内容

1. **多智能体协同核心架构**：构建角色分明、边界清晰的运维数字专家矩阵，通过统一调度与现有工具平台深度对接。
2. **故障应急场景优化**：融合大模型+小模型协同联动，小模型负责算法提纯，大模型负责逻辑推理，提升根因定界与处置效率。
3. **路径实现验证与场景深化**：在可行性验证基础上，持续迭代平台功能，推动AI+运维场景深度融合。

项目计划于2026年8月至12月分三个里程碑完成基础版本上线、对接试运行与验收推广。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 关键名词关联

- [[Harness工程]]：AI工程化新范式，为智能体构建运行环境与约束闭环。
- [[LangChain]]：开源框架，用于连接大语言模型与外部资源/工具。
- [[DeepAgents]]：基于LangChain的深层智能体框架，支持复杂长周期任务。

## 风险与应对

项目面临智能体安全风险、技术迭代风险、人力投入不足及算力不足等挑战，已制定安全栅栏、优先选用稳定技术路线、以联盟人员为主采购外部服务、选择试点系统验证等应对措施。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
