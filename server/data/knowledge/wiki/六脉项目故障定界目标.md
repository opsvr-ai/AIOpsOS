# 六脉项目故障定界目标

**六脉项目**以提升[[故障取证与根因定界]]效率为核心目标之一，明确提出支撑[[21+6关键业务系统]]故障定界能力的实现。项目旨在通过AI多智能体技术，系统性地解决传统运维模式下故障定位慢、定界难的问题。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 目标分解

- **覆盖范围**：优先覆盖21+6关键业务系统，后续逐步扩展到所有重要生产系统。
- **核心指标**：缩短故障根因定界时间，将原本需要数小时的人工分析缩短至分钟级，提升定界准确性。
- **实现路径**：基于“大模型+小模型”协同机制，小模型负责数据提纯，大模型负责逻辑推理，快速输出故障处置建议。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 与应急协同的关系

故障定界目标与[[fault-emergency-agent-collaboration|应急协同]]紧密耦合：快速定界是高效应急的基础，项目通过引入运维数字专家智能体，实现多岗位信息自动同步，提升整体处置效率。

## 当前进展

项目尚处于起步阶段，计划在2026年8月完成基础版本上线，并在2026年第四季度进行核心场景验证，届时将对21+6系统的故障定界能力进行实测。
