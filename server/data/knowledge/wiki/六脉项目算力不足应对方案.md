# 六脉项目算力不足应对方案

## 问题描述

六脉项目采用，对算力供给能力和并发处理能力有极高要求。项目风险分析明确指出，联盟现有算力规划在近年内无法满足满血运行需求。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 应对策略

### 1. 试点先行

不追求全面铺开，选择[[21+6关键业务系统|21+6关键业务系统]]中的部分系统作为试点，进行路径探索和能力验证。^[raw/公司级创新申报材料-生产运维部_v0407-5.md] 通过试点积累经验，逐步优化算力消耗。

### 2. 分阶段算力规划

结合[[六脉项目2026年工作计划详解|2026年工作计划]]，分阶段评估算力需求：
- **第一阶段（基础版本）**：采用轻量级模型+[[大小模型协同|大小模型协同]]，减小大模型调用频率。
- **第二阶段（场景对接）**：根据试点反馈，评估是否需要扩容算力。
- **第三阶段（推广）**：在验证可行后，向上申请专项算力预算。

### 3. 技术优化

利用[[Harness工程理念在六脉项目中的应用|Harness工程]]中的约束机制，限制非关键任务的推理开销；通过[[智能体安全栅栏|智能体安全栅栏]]控制智能体行为，避免不必要的计算资源浪费。

### 4. 混合模式

探索的模式，将算力密集环节集中在必要时刻，日常监控使用轻量级模型。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 预期效果

通过以上组合策略，预计可在现有算力约束下完成[[六脉项目故障定界目标|故障定界目标]]的技术验证，并为后续扩大规模提供可行的扩容依据。
