# 六脉项目大小模型协同机制

在[[六脉项目故障取证与根因定界]]场景中，六脉智能运维数智平台采用了“**大模型 + 小模型**”协同联动模式，以提升故障根因定位的准确性与效率。该模式是平台核心创新能力之一。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 分工与协作

- **小模型**：负责算法提纯与快速筛选。在故障发生时，小模型优先处理海量原始运维数据，快速提炼出关键的异常指标、日志片段等，压缩信息量并降低后续分析的噪声。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
- **大模型**：侧重逻辑推理与根因研判。在接收到小模型提炼的关键数据后，大模型利用其强大的上下文理解与推理能力，综合多维度信息，辅助判断故障根本原因，并生成处置建议。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]

## 应用价值

- **缩短定位时间**：大小模型分工避免了让大模型直接处理海量原始数据导致的“浅层”分析，可快速缩小排查范围，提升根因定界的效率。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
- **降低算力消耗**：小模型承担数据预处理环节，减少大模型需要处理的信息量，在有限算力资源下实现更高效的诊断。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
- **支撑多系统运维**：该机制可并行服务于多个业务系统，为[[21+6关键业务系统]]的故障定界目标提供技术基础。^[raw/公司级创新申报材料-生产运维部_v0407-5.md]
