## OpsOS 项目结构优化与 Docker-Compose 部署

平台命名为 **OpsOS** (运维智能操作系统)，体现其作为运维领域 AI 操作系统的定位——集感知、决策、执行、进化于一体的智能体基座。

### 二、核心目录说明

#### 1. server/ — 后端服务
- **agent/**：多智能体引擎核心。包含主智能体 LangGraph 状态图、子智能体工厂（alert_analysis、root_cause 等）。每个节点逻辑独立，便于测试与扩展。
- **core/**：平台通用能力。`tool_manager` 统一管理 Skill/MCP 工具的热加载；`mcp_connector` 维护 MCP 会话池；`memory_core` 提供记忆检索与沉淀；`skill_executor` 提供安全沙箱执行。
- **integrations/**：外部数据接入。`kafka_consumer` 负责消费告警事件；`data_adapters/` 封装 Prometheus、Zabbix 等系统的拉取适配器。
- **workers/**：Celery 异步任务与 RedBeat 定时调度。

#### 2. web/ — 前端应用
- **components/chat/**：对话核心组件，包括意图卡片、规划卡片、执行卡片、安全确认卡片、参数收集表单等，全部按需加载。
- **features/**：每个页面独立模块，路由级代码拆分，保证首屏极快。
- **theme.ts**：Ant Design Token 深度定制，塑造极简科技风深色主题。

#### 3. knowledge-base/ — LLM-Wiki 知识库
遵循三层架构：
- `raw/`：原始文档（不可变），支持拖拽上传、从告警自动生成。
- `wiki/`：AI 编译的知识页面（Markdown），提供全文检索与交叉引用。
- `schema/`：人工定义的质量标准与格式规范。

#### 4. deploy/ — 部署配置
提供生产级 `docker-compose.yml` 与环境变量模板，一键启动全部服务。

### 三、Docker-Compose 一键部署配置

#### 环境变量模板 (`.env.example`)

```
# PostgreSQL
POSTGRES_DB=opsos
POSTGRES_USER=opsos
POSTGRES_PASSWORD=changeme
POSTGRES_HOST=db

# Redis
REDIS_URL=redis://redis:6379

# Kafka
KAFKA_BOOTSTRAP_SERVERS=kafka:9092

# OpsOS 后端
SECRET_KEY=change-me-to-a-random-string
JWT_ALGORITHM=HS256
DATABASE_URL=postgresql+asyncpg://opsos:changeme@db:5432/opsos
CELERY_BROKER_URL=redis://redis:6379/0
CELERY_RESULT_BACKEND=redis://redis:6379/1
LLM_API_KEY=sk-xxx
LLM_BASE_URL=https://api.openai.com/v1
```

#### `docker-compose.yml`

```yaml
version: '3.9'

services:
  # ---------- 基础设施 ----------
  db:
    image: pgvector/pgvector:pg15
    container_name: opsos-db
    environment:
      POSTGRES_DB: ${POSTGRES_DB:-opsos}
      POSTGRES_USER: ${POSTGRES_USER:-opsos}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:-changeme}
    volumes:
      - db_data:/var/lib/postgresql/data
      - ./deploy/init-db.sql:/docker-entrypoint-initdb.d/init.sql
    ports:
      - "5432:5432"

  redis:
    image: redis:7-alpine
    container_name: opsos-redis
    ports:
      - "6379:6379"

  kafka:
    image: confluentinc/cp-kafka:7.5.0
    container_name: opsos-kafka
    hostname: kafka
    environment:
      KAFKA_NODE_ID: 1
      KAFKA_PROCESS_ROLES: 'broker,controller'
      KAFKA_LISTENERS: 'PLAINTEXT://:9092,CONTROLLER://:9093'
      KAFKA_ADVERTISED_LISTENERS: 'PLAINTEXT://kafka:9092'
      KAFKA_CONTROLLER_LISTENER_NAMES: 'CONTROLLER'
      KAFKA_LISTENER_SECURITY_PROTOCOL_MAP: 'CONTROLLER:PLAINTEXT,PLAINTEXT:PLAINTEXT'
      KAFKA_OFFSETS_TOPIC_REPLICATION_FACTOR: 1
      CLUSTER_ID: 'MkU3OEVBNTcwNTJENDM2Qk'
    ports:
      - "9092:9092"

  # ---------- 后端服务 ----------
  api:
    build:
      context: ./server
      dockerfile: Dockerfile
    container_name: opsos-api
    depends_on:
      - db
      - redis
      - kafka
    environment:
      DATABASE_URL: ${DATABASE_URL}
      REDIS_URL: ${REDIS_URL}
      KAFKA_BOOTSTRAP_SERVERS: ${KAFKA_BOOTSTRAP_SERVERS}
      SECRET_KEY: ${SECRET_KEY}
      LLM_API_KEY: ${LLM_API_KEY}
      LLM_BASE_URL: ${LLM_BASE_URL}
    volumes:
      - knowledge_data:/app/knowledge-base   # LLM-Wiki 知识库
    ports:
      - "8000:8000"
    command: uvicorn src.main:app --host 0.0.0.0 --port 8000

  worker:
    build:
      context: ./server
      dockerfile: Dockerfile
    container_name: opsos-worker
    depends_on:
      - db
      - redis
      - kafka
    environment:
      DATABASE_URL: ${DATABASE_URL}
      REDIS_URL: ${REDIS_URL}
      KAFKA_BOOTSTRAP_SERVERS: ${KAFKA_BOOTSTRAP_SERVERS}
      SECRET_KEY: ${SECRET_KEY}
      LLM_API_KEY: ${LLM_API_KEY}
      LLM_BASE_URL: ${LLM_BASE_URL}
    volumes:
      - knowledge_data:/app/knowledge-base
    command: celery -A src.workers.celery_app worker -l info -c 4

  beat:
    build:
      context: ./server
      dockerfile: Dockerfile
    container_name: opsos-beat
    depends_on:
      - db
      - redis
      - kafka
    environment:
      DATABASE_URL: ${DATABASE_URL}
      REDIS_URL: ${REDIS_URL}
      KAFKA_BOOTSTRAP_SERVERS: ${KAFKA_BOOTSTRAP_SERVERS}
      SECRET_KEY: ${SECRET_KEY}
      LLM_API_KEY: ${LLM_API_KEY}
      LLM_BASE_URL: ${LLM_BASE_URL}
    command: celery -A src.workers.celery_app beat -l info --scheduler redbeat.RedBeatScheduler

  kafka-consumer:
    build:
      context: ./server
      dockerfile: Dockerfile
    container_name: opsos-kafka-consumer
    depends_on:
      - db
      - redis
      - kafka
    environment:
      DATABASE_URL: ${DATABASE_URL}
      REDIS_URL: ${REDIS_URL}
      KAFKA_BOOTSTRAP_SERVERS: ${KAFKA_BOOTSTRAP_SERVERS}
      SECRET_KEY: ${SECRET_KEY}
    command: python -m src.integrations.kafka_consumer

  # ---------- 前端 ----------
  frontend:
    build:
      context: ./web
      dockerfile: Dockerfile
      args:
        VITE_API_BASE_URL: http://localhost:8000
    container_name: opsos-frontend
    ports:
      - "3000:80"
    depends_on:
      - api

volumes:
  db_data:
  knowledge_data:
```

#### 后端 Dockerfile 示例 (`server/Dockerfile`)

```dockerfile
FROM python:3.11-slim

WORKDIR /app

RUN apt-get update && apt-get install -y --no-install-recommends \
    gcc libpq-dev && rm -rf /var/lib/apt/lists/*

COPY pyproject.toml poetry.lock ./
RUN pip install --no-cache-dir poetry && poetry config virtualenvs.create false \
    && poetry install --no-dev --no-interaction --no-ansi

COPY . .

EXPOSE 8000
CMD ["uvicorn", "src.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

#### 前端 Dockerfile 示例 (`web/Dockerfile`)

```dockerfile
FROM node:20-alpine AS build
WORKDIR /app
COPY package.json pnpm-lock.yaml ./
RUN npm install -g pnpm && pnpm install --frozen-lockfile
COPY . .
ARG VITE_API_BASE_URL
RUN pnpm build

FROM nginx:stable-alpine
COPY nginx.conf /etc/nginx/conf.d/default.conf
COPY --from=build /app/dist /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

#### Nginx 配置 (`web/nginx.conf`)

```nginx
server {
    listen 80;
    server_name _;
    root /usr/share/nginx/html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://api:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location /ws/ {
        proxy_pass http://api:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

### 四、快速启动

```bash
# 1. 克隆项目
git clone https://github.com/your-org/opsos.git && cd opsos

# 2. 配置环境变量
cp deploy/.env.example deploy/.env
vim deploy/.env   # 修改必要密钥

# 3. 一键启动
docker-compose -f deploy/docker-compose.yml up -d
```

访问 `http://localhost:3000` 即可使用 OpsOS，前端自动代理 API 请求到后端。

这套项目结构与部署方案，完全贴合 OpsOS 作为运维 AI 操作系统的定位：模块边界清晰、可扩展、支持一键部署，同时将 LLM-Wiki 知识库体积持久化，保障知识资产安全。
