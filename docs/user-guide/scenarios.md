# 场景运维

## 什么是场景

场景是预设的运维任务模板，将特定的工具和智能体组合在一起，形成可复用的运维工作流。

例如：
- **日常巡检** — CPU/内存/磁盘/网络指标检查
- **故障排查** — 根据告警信息自动定位根因
- **变更发布** — 灰度发布、回滚操作

## 创建场景

1. 进入「运维中心 → 场景运维」
2. 点击「新建场景」
3. 填写场景名称和描述
4. 关联所需的工具（Skill / MCP）
5. 关联负责的子智能体
6. 保存

## 使用场景

在对话中，通过以下方式调用场景：

- 点击输入框上方的场景快捷标签
- 输入 `/` 唤出命令面板，选择场景
- 直接在对话中描述场景名称

智能体会根据场景配置加载对应的工具和子智能体。

## 场景关联

| 关联项 | 说明 |
|--------|------|
| 工具 (Tools) | 场景可用的 Skill 和 MCP 工具列表 |
| 子智能体 (SubAgents) | 负责执行场景任务的智能体 |
| 定时任务 (Cron) | 周期性自动执行场景 |
| 触发规则 (Trigger) | 告警触发时自动关联场景 |
