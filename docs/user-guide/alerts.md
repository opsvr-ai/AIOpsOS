# 告警中心

## 告警来源

告警通过以下渠道接入：

- **Kafka 消息队列** — 外部监控系统推送告警事件
- **Webhook 回调** — Prometheus / Grafana / 自定义系统回调
- **手动创建** — 运维人员手动录入

## 告警状态流转

告警遵循严格的状态机：

```
pending → analyzing → awaiting_review → confirmed/dismissed → closed
```

| 状态 | 说明 |
|------|------|
| `pending` | 新告警，等待处理 |
| `analyzing` | 智能体正在自动分析 |
| `awaiting_review` | 分析完成，等待人工确认 |
| `confirmed` | 已确认为有效告警 |
| `dismissed` | 已忽略（误报/不重要） |
| `closed` | 已处理完毕，归档 |

## 自动分析

告警进入系统后，触发规则匹配会：

1. 匹配关联的场景和子智能体
2. 自动创建分析会话
3. 调用智能体进行根因分析
4. 结果写入分析结果字段
5. 状态变更为待审核

## 告警操作

- **查看详情** — 点击告警卡片展开完整信息
- **手动分析** — 对未自动分析的告警手动触发分析
- **确认** — 标记为有效告警，可选择将分析结果沉淀到知识库
- **驳回** — 标记为误报或无需处理
- **批量操作** — 勾选多条告警进行批量确认/驳回
- **筛选** — 按状态、时间范围、主机名筛选
