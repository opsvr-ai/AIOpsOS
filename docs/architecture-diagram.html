Reference · 设计模式

六大设计模式

经过验证的架构决策，支撑平台的稳定与可扩展

01
Model Factory
统一 LLM 接口抽象 — get_default_model() / get_model_for_agent()，支持 Anthropic / DeepSeek / OpenAI 兼容，Redis 缓存 (TTL 300s)
02
Idempotent Insert
pg_insert(...).on_conflict_do_nothing() 幂等写入，避免 try/except 裸插入，确保数据一致性
03
Lazy Model Init
Agent __init__ 接受可选 model 参数，async _get_llm() 懒加载，避免启动时阻塞
04
Space Context
get_current_space() 从 Agent 上下文继承 space_id，确保 DB 记录归属正确的多租户空间
05
HITL 人工中断
asyncio.Event 异步机制 — Agent → InterruptManager → SSE 通知前端 → 用户响应 → Agent 恢复
06
Tool 安全分级
parallel-safe (并发) / sequential (串行) / destructive (HITL 确认)，ToolDispatcher 自动调度
