## 企业级平台设置：品牌、LDAP 与细粒度权限

在控制中心，我们将品牌定制、LDAP 集成和权限管理统一到**系统管理**和**身份与权限**模块，管理员无需离开平台即可完成企业级配置。所有界面延续极简风格，复杂配置通过步骤引导或抽屉面板完成。

---

### 一、自定义品牌（名称与 Logo）

**位置**：控制中心 → 系统管理 → 品牌设置  
**界面元素**：
- **平台名称**：文本输入框，修改后实时预览顶部 Logo 区域文字。
- **Logo 上传**：拖拽上传区，支持 PNG/SVG，建议尺寸 32×32 或 200×50。上传后自动裁剪，深/浅色主题各一套。
- **Favicon**：同 Logo 可同步生成。
- **预览**：右侧显示模拟的登录页和主界面效果。

**技术实现**：
- 配置保存在数据库 `system_config` 表（键值对），或独立的 `branding` JSON 文件，支持热更新。
- 前端通过 Context 读取配置，动态替换 `document.title`、`link rel="icon"`、登录页 Logo 等。

---

### 二、LDAP 集成

**位置**：控制中心 → 身份与权限 → LDAP 配置  
**设计思路**：提供向导式配置，测试连接后自动同步用户。

**四步配置向导**：

1. **服务器信息**：填写 LDAP URL、基础 DN、管理员 DN 及密码。
2. **用户映射**：定义对象类（通常 `person`）、登录名属性（如 `sAMAccountName`）、邮箱属性、显示名属性。
3. **同步策略**：手动/定时同步（Cron 表达式）、首次导入是否开启、用户状态（创建后默认启用/禁用）。
4. **测试与预览**：输入一个测试用户名，验证是否能正确查询并显示映射后的属性，确认无误后保存。

**权限承接**：LDAP 用户首次登录后自动创建本地账户，并可根据 LDAP 组映射角色（需要在 RBAC 中预先设置组规则）。

**界面组件**：每步一个表单卡片，底部导航按钮，连接错误友好提示。

### 三、细粒度权限（RBAC）

**位置**：控制中心 → 身份与权限 → 角色与权限

**三要素**：
- **用户**：本地用户或 LDAP 同步用户。
- **角色**：自定义角色，如“运维管理员”、“观察者”、“工具注册者”等。
- **权限**：基于资源+操作的最小单位。

**权限模型设计**：

采用**资源-操作**对形式，允许前端动态展示/禁用。

| 资源类别 | 示例资源 | 操作 |
|---------|---------|------|
| 场景 | scenarios | view, create, edit, delete, execute |
| 工具 | tools | view, register, approve, disable |
| 调度 | schedules | view, create, edit, delete, trigger |
| 触发器 | triggers | view, create, edit, delete |
| 数据接入 | data_sources | view, config, health_check |
| 渠道 | channels | view, create, edit, disable |
| 智能体 | agents | view, create, edit, test |
| 知识库 | knowledge | view, upload, delete |
| 系统设置 | system | view, edit (品牌、LDAP、备份等) |
| 审计日志 | audit | view |
| 对话 | chat | execute, stop (所有场景执行权限) |
| 用户管理 | users | view, create, edit, delete |

**权限数据库设计**：
- `permissions` 表：id, resource, action, description。
- `roles` 表：id, name, description。
- `role_permissions`：角色-权限多对多。
- `user_roles`：用户-角色多对多。

**界面设计**：
- **角色列表**：卡片式网格，显示角色名、描述、用户数。
- **编辑角色**：右侧抽屉，分为权限矩阵（勾选资源操作）和成员列表（可增删用户）。
- **用户详情页**：在用户列表中点击用户，显示其当前角色、LDAP 信息、可快速切换角色。
- **权限矩阵视图**：以表格形式，行为资源，列为操作（查看/创建/编辑/删除等），单元格为复选框，支持批量勾选“整行”或“整列”，极大简化配置。

**安全特性**：
- 无权限的导航菜单项自动隐藏或灰显。
- API 层强制校验，前端仅为提升体验，后端为最终安全防线。
- 操作审计日志记录所有权限变更。

---

### 四、与现有模块的结合

- 所有以上配置均在**控制中心**下，不干扰运维操作。
- 导航栏仅当用户具有 `system` 或 `users` 权限时才显示“系统管理”和“身份与权限”菜单。
- LDAP 同步的用户默认分配“观察者”基础角色，仅可查看对话历史，需管理员手动提权。
- 自定义品牌立即反映在登录页、导航栏 Logo、浏览器标题，增强企业归属感。

通过这三项企业级能力的补充，平台具备了完整的商用就绪度，既保持了极简交互，又提供了坚固的安全与权限底座。
