## 极简对话界面整合：场景、附件与执行控制

为了让对话界面承载所有运维场景，同时保持极简，我们将**场景切换**、**附件上传**、**终止执行**等功能隐式化，仅在需要时轻量展开。核心交互围绕“命令触发”和“上下文感知”两条路径，界面元素至简，但能力完整。

---

### 一、场景启动方式设计

提供 **两种无感启动模式**，用户无需离开对话界面：

#### 1. 斜杠命令 (Slash Command)  
- 在输入框输入 `/`，即刻弹出轻量命令面板（类似 Notion/IDE 风格）。
- 面板列出所有可用场景及描述，支持快速搜索筛选。
- 选择后，自动填充智能体输入，并提示可选的参数占位符。
- 例如输入 `/巡检` → 发送给智能体，开启巡检流程。

#### 2. 快捷场景标签 (Pills)  
- 对话框顶部下方设一行**可水平滚动的微型标签**（仅图标+名称，悬停显示完整名称）。
- 默认只展示 3~4 个最常用场景，其余折叠在“更多...”按钮中。
- 点击标签立即发送对应场景指令。
- 支持用户个人常用场景自定义（记忆最近使用或收藏）。

两种方式配合，既满足高级用户快速输入，也符合鼠标操作习惯，且占据视觉空间极小。

---

### 二、附件上传功能

附件上传按钮隐藏在输入框右侧，仅以一个回形针图标 📎 存在，无文字。

**交互流程**：
1. 点击图标或拖拽文件到对话区。
2. 弹出**轻量上传卡片**，显示文件名、大小、上传进度。
3. 上传完成后，卡片内生成摘要（如图片缩略图、日志前几行预览），并允许移除。
4. 提交消息时，附件信息将以结构化上下文注入智能体（例如“用户上传了日志文件 xxx.log，内容摘要…”），用户亦可手动添加文字说明。

**设计要点**：
- 不上传即时不可见，不上传完成不影响消息编辑。
- 附件数量限制（如 3 个）且总大小限制（如 20MB），超出给出提示。
- 支持常见运维文件类型：`.log`, `.txt`, `.json`, `.png`, `.tar.gz` 等。

---

### 三、对话终止与执行控制

当智能体正在执行任务时，**发送按钮变为“停止”按钮**（设计为红色圆形停止图标），点击后弹出二次确认（是否终止当前任务），确认后前端调用 API 取消会话，后端通过 `interrupt` 或 `cancel` 机制中止执行，并保留已生成的部分结果。

同时在对话顶部状态栏以小文字显示“智能体运行中...”并附带**动态加载条**（细光条），用户可随时感知进度。点击状态栏也可触发停止。

### 四、整合后的对话界面布局（更新）

```
┌─────────────────────────────────────────────────────┐
│ [运维中心 / 对话]    当前场景：告警分析 ▾   [停止⏹] │ ← 顶栏：可切换场景，停止按钮显隐
├─────────────────────────────────────────────────────┤
│                                                     │
│   [场景快捷标签：🔍告警分析 📋日常巡检 🔧故障定界 ...] │ ← 可滚动
│                                                     │
│   ╔══════════════════════════════════════╗          │
│   ║ 💬 对话消息流                        ║          │
│   ║   - 意图识别卡片                     ║          │
│   ║   - 任务规划卡片                     ║          │
│   ║   - 执行过程卡片                     ║          │
│   ║   - 安全确认卡片 / 参数收集表单      ║          │
│   ║   - 最终回答 + 引用                 ║          │
│   ╚══════════════════════════════════════╝          │
│                                                     │
├─────────────────────────────────────────────────────┤
│ [📎 上传文件] [图片小图标]                           │ ← 附件区
│ [输入框...  / 触发命令提示]            [发送⏎/停止⏹] │
└─────────────────────────────────────────────────────┘
```

- 顶栏的“当前场景”是一个下拉框，可选择场景并自动发送初始提示，或通过命令/标签触发后自动切换。
- 停止按钮仅在智能体会话进行中可见，平时为发送按钮。
- 附件信息若存在，会在输入框上方以微型标签显示已上传文件，可独立删除。

### 五、与现有卡片组件的和谐关系

- 当用户发送场景指令后，对话流中像之前一样出现意图卡片、规划卡片等，不受影响。
- 安全确认卡片和参数收集表单继续作为内联卡呈现，与新增的附件上传、命令面板无冲突，它们都在同一对话流中产生。
- 对话终止时，若已有部分结果，会在对话流最后追加一条“⏹️ 用户终止了执行”的系统消息，已生成的卡片保留，方便后续排查。

---

### 六、前端实现关键点

- **命令面板**：使用 Ant Design 的 `AutoComplete` 或自定义浮层，监听 `/` 键触发，数据来自场景列表 API。
- **场景标签**：Ant Design `Tabs` 或自定义 `ScrollablePills`，支持横向滚动和“更多”下拉。
- **附件上传**：Ant Design `Upload` 组件，`showUploadList` 定制，配合后端临时上传接口。
- **停止按钮**：基于会话状态（`session.status === 'running'`）切换按钮样式和功能，通过全局 Zustand 状态管理。
- **消息流**：`react-virtuoso` 虚拟列表确保性能，并可灵活插入各类型卡片。

通过以上整合，对话界面成为了**真正的运维指挥中心**——无需跳转，即可启动全场景、上传证据、控制执行，且界面依然保持极简科技感。所有操作以最少的视觉干扰呈现，体现了“少即是多”的设计哲学。
